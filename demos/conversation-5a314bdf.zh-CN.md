# Demo：对话 5a314bdf-c17a-4bb0-9201-dcf8697cb4d2

这个 demo 使用对话 `5a314bdf-c17a-4bb0-9201-dcf8697cb4d2` 中讨论过的真实产品想法。

## 用户想法

```text
我想把这个 skill 开源，并做一个线上版本进行收费。首次免费，后续按次收费。
```

## Skill 输出

### 一句话判断

按当前描述，失败风险大约是 75%-85%，信心：中等。

风险不是这个 skill 没有价值，而是“开源 skill + 线上按次收费版本”这个商业模式很容易死在：付费需求弱、差异化不清楚、分发困难、免费版本和付费版本冲突。

如果重新定位成“开源 skill 做分发，付费产品卖决策级创业诊断报告”，失败风险可以降到大约 45%-60%。

### 最可能失败链条

```text
No Market Need -> Distribution Vacuum -> Competition / Commodity Trap -> Unit Economics
```

翻译成人话：

```text
大家觉得这个 skill 有意思
-> 但不一定愿意付费
-> 开源版本已经够用
-> 线上版差异化不强
-> 获客成本超过按次收入
-> 项目变成小工具，而不是可持续业务
```

### 风险评分

- Demand: 4/5 - 用户可能喜欢“创业失败诊断”，但愿不愿意为一次报告付费还没有验证。
- Product/Tech: 2/5 - 做网页和支付不是难点，难点是诊断质量、可信度和持续价值。
- Competition: 4/5 - 这个概念可以被 prompt、GPT、Notion 模板或另一个 agent skill 快速复制。
- Distribution: 5/5 - 最大风险是如何触达 founder，并让用户反复使用。
- Unit Economics: 3/5 - 按次收费理论上毛利不错，但低客单价可能被模型成本、支付手续费和获客成本吃掉。
- Cash/Runway: 2/5 - 项目可以轻量启动，现金消耗可控。
- Operations/Regulation/Team: 2/5 - 法规风险低，但不能把产品包装成投资、法律或商业成功保证。

### 关键 Red Flags

- 开源 skill 和付费 web 版本之间的价值边界不清楚。
- “首次免费，后续按次收费”可能带来好奇用户，但转化弱，因为创业诊断不一定是高频需求。
- 付费用户画像还不够具体：indie hacker、founder、投资人、产品团队、企业创新团队的预算和期望完全不同。

### 需要的反证

- 20 个提交创业想法的人里，至少 3 个愿意为完整诊断付费。
- 开源 repo 能带来真实线索，而不只是 GitHub stars。
- 付费产品明显优于用户复制 skill 到本地自己跑。
- 用户反馈不是“挺有意思”，而是“这个报告改变了我的 build / kill / pivot 决策”。

### 7-14 天验证计划

1. 做一个 landing page，标题可以是：`在开工前，先诊断你的创业想法会怎么死。`
2. 设置三个版本：
   - 免费：基础失败链条
   - 低价：风险评分 + red flags
   - 高价：完整报告 + 验证计划 + kill criteria
3. 前 10 份报告手动交付，不要一开始做完整自动化。
4. 在 founder 聚集的渠道发布 5 个公开诊断案例。
5. 衡量提交率、付费转化率，以及用户是否会要求第二次诊断。

### Kill Criteria

- 如果 100 个访问者带来少于 5 个创业想法提交，停止或转向。
- 如果 20 个提交里少于 3 个付费用户，停止或转向。
- 如果免费报告互动不错，但付费转化低于 5%，停止或转向。
- 如果用户只说“有意思”，但没有用报告改变决策，停止或转向。
- 如果获客成本超过单次付费收入的 3 倍，停止或转向。

### Rebuild Angle

不要把它定位成：

```text
开源 skill 的线上付费版
```

更好的定位是：

```text
Startup Autopsy Simulator
```

或者：

```text
创业想法死亡风险报告生成器
```

开源 skill 应该是分发渠道。付费产品应该卖“决策级报告”：失败链条、PMF 风险、CAC/LTV 风险、red flags、验证计划和 kill criteria。

用户不是在为“调用一次 AI”付费，而是在为一个更好的 build / kill / pivot 决策付费。
